# Техническое Задание: Платформа управления персоналом и заказами (аналог Staffy)

## 1. Общее описание проекта
Проект представляет собой SaaS-платформу для автоматизации процессов найма временного персонала (грузчики, разнорабочие и др.) через сеть Telegram-ботов. Система объединяет управление заказами, базу сотрудников (CRM) и инструменты финансовой отчетности в едином веб-интерфейсе для администраторов и менеджеров.

---

## 2. Основные модули системы

### 2.1. Панель управления (Dashboard)
- **Список заказов**: Отображение всех активных, запланированных и завершенных заказов.
- **Фильтрация**: Сортировка по городам (ботам), статусам оплаты и типам работ.
- **Индикаторы**: Цветовое кодирование срочности заказов и задолженностей клиентов.

### 2.2. Управление заказами
- **Конструктор заказа**:
    - Выбор Telegram-бота для трансляции.
    - Выбор шаблона заказа (для повторяющихся работ).
    - Ввод данных клиента (Имя, Телефон).
    - Адрес объекта (Район, Улица, Дом, Офис).
    - Требования: Дата, время, количество человек, тип работ.
    - Чек-листы (например, "Трезвость", "Спецодежда").
- **Логика рассылки**: Возможность немедленной публикации в Telegram-бот или отложенного старта.
- **Список откликов**: Просмотр сотрудников, нажавших кнопку «Принять» в боте, с возможностью назначения на смену.

### 2.3. CRM сотрудников (База)
- **Профили**: ФИО, рейтинг, контакты (один или несколько номеров).
- **Тегирование**: Назначение ролей и категорий (например, «грузчик», «стройплощадка», «черный список»).
- **Статусы**: «В работе», «Свободен», «Забанен».
- **История**: Логирование всех выполненных заказов сотрудником.

### 2.4. Единая лента контактов (Модерация)
- Просмотр всех пользователей, запустивших ботов.
- Одобрение новых анкет.
- Инструменты бана и перемещения в черный список.

### 2.5. Финансы и Бухгалтерия
- Учет выручки и расходов (выплаты сотрудникам) по каждому заказу.
- Автоматический расчет прибыли.
- Отчеты за период: Выгрузка данных по обороту и прибыли.

### 2.6. Управление ботами и правами
- Регистрация новых Telegram-ботов через API токены.
- Статистика подписчиков по каждому боту.
- Создание учетных записей менеджеров с разграничением прав доступа.

---

## 3. Технические требования

### 3.1. Технологический стек (рекомендуемый)
- **Backend**: Python (FastAPI/Django) или Node.js.
- **Frontend**: React/Next.js с использованием современных UI-библиотек (Tailwind CSS, shadcn/ui).
- **Бот-движок**: aiogram (Python) или Telegraf (Node.js) для работы с Telegram API.
- **База данных**: PostgreSQL (реляционные данные) + Redis (очереди сообщений).
- **Инфраструктура**: Docker для контейнеризации, Nginx, SSL сертификаты.

### 3.2. Интеграция с Telegram
- Использование **Inline-кнопок** для взаимодействия с сотрудниками.
- Система **Webhooks** для мгновенной обработки откликов.
- Групповые рассылки и индивидуальные уведомления (например, напоминание о смене за час).

---

## 4. Требования к UI/UX
- **SaaS-эстетика**: Чистый, современный интерфейс ("Dashboard style").
- **Адаптивность**: Платформа должна быть удобной как на десктопе, так и на планшетах для оперативной работы менеджера в "полевых" условиях.
- **Скорость**: Быстрый поиск по базе сотрудников (ElasticSearch или оптимизированный SQL).

---

## 5. Безопасность
- Шифрование персональных данных.
- Двойная аутентификация для администраторов.
- Разделение ответственности: менеджер видит только свои боты/заказы (опционально).
